Week 4 Blog Post - Make it Beautiful

Published on 24th October 2017

The rabbit hole that is CSS (Cascading Style Sheets) has been opened up to me this week. Below is a screenshot of my site before CSS has been applied.

It being a declarative language makes it simple to write. However due to their being a lot of properties to learn, knowing what property to use to achieve the desired result is proving to be the greatest challenge for writing CSS. To help with this there is a CSS Reference which lists all the possible properties and then provides further information on the possible values that can be applied.

Due to the inconsistency between web browsers default stylesheets, developers are required to reset elements that are styled by the default stylesheets. Two different methods can be applied:

  • Reset which removes all styling on all elements
  • Normalize which removes styling on elements that are inconsistent between web browsers and adds a default styling.

I’ve been recommended to use the normalize method. The stylesheet for which can be found here.

Having readable content on a webpage is essential and websites now have to accommodate a wide range of conditions where their website may be viewed. Whether the page is being viewed on a smartphone whilst the user is walking along or on a desktop computer in an office environment all content needs to be readable. There are recommended web safe fonts that are readable and the majority of browsers and OSes support. The next few weeks will be full of experimentation so expect some interesting looking designs.

Week 5 Blog Post - Boxing it Up

Published on 12th November 2017

Block elements, unsurprisingly, are contained within boxes. Having played around with the developer settings in Chrome I knew that elements had boxes around them but did not really understand their purpose or how to implemHTML & CSS by Jon Duckett, this was cleared up for me and I was able to effectively use the wrapper to center my website. This mainly had the effect of improving the readability of my site.

Humans find it hard to read long lines of text, hence why programmers indent and format their code so it takes up many lines.

Another factor that affects readability is the font being used. OSes have some fonts packaged with them but not all of them are the most suitable for the web. Google Fonts allows you to choose from over 800 fonts to be used on your webpage and even provides the code required to refer to them in your CSS and HTML documents. Though having fonts that you can fall back on incase this service goes down is always a good idea. This allowed me to have a different font for my main page heading and my paragraphs. The main page heading got a font that was more readable when small whilst the heading font was chosen for standing up.

I spent quite a large amount of time choosing which font to use and whilst I think as I gather more experience with Web Development I will learn what fonts are good to use, having a standardised set of approved fonts might aid accessibility on the web.

I was able to style my table that I produced in week 1 so now that is more useful for reference purpose as it is more legiable.

I have been enjoying my time with CSS and do not see myself coming out of this rabbit hole for a very long time.
